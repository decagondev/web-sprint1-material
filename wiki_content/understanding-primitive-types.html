Overview

Primitive types are the most basic types of data used in Javascript, i.e. they can not be broken down into smaller parts that are still recognized by the language. The most widely used primitive types include strings, numbers, and booleans.

Here are some examples and descriptions of each:


"I am a sentence" // this is a string of text, or string, delimited by double-quotes.
'I am also a sentence' // another string, in this case using single-quotes. In JavaScript single and double-quotes work the same, but you can not use both forms on the same string.
53 // this is a number. Whole numbers are commonly called integers, but they are still of type number in JavaScript.
12.34 // another number. Numbers with a decimal point are often called floating-point numbers.
true // this is a Boolean. Booleans are either true or false.
false // another Boolean.

Some other primitive types represent an absence of value. These include undefined, null, and NaN.

Here are some examples of these:


undefined // this is a special value and also a type, which signifies a complete absence of a value. If a value is not given, the system defaults to undefined.
null // another special value to signify absence. Null is used to say that the developer specifically wants this value to be absent.
NaN // "Not A Number", a value that represents things like zero divided by zero.

Identifying String Values

Let’s return to your work on the web site that provides information on all of the active clubs at Washington Community Center. Towards the beginning of the project, you want to first teach your students about the different primitive types of values that they should expect to handle in their web application: strings, numbers, booleans, and undefined. You start with introducing strings and remind the students of an example of a string that was logged to the console earlier:

console.log("The Improv Club meets every Tuesday and Thursday in the theater.")

Right now, this is a very long string, but eventually, you would like to use the pattern of this string as a template for three smaller strings: a club name, meeting dates, and meeting location. You ask your students to break down each of these into their individual string values:

  • "The Improv Club"
  • "Tuesdays and Thursdays"
  • "Theater"

Identifying Number Values

Next, you want to draw attention to some of the number values they would expect to see in this project. Here are a few that come to mind:

  • Number of students currently enrolled in each club
  • Maximum number of students allowed in each club
  • Time the club starts

This last one is tricky because it involves time. If you typed a time, say 3:10 in the console, you would get a syntax error because Javascript doesn’t recognize this format (number, colon, number). If you typed “3:10” with quotes, Javascript would treat it as a string. This could work, but it might be more difficult to later code for an option for your students to search a club by “time met”. Lastly, you could treat time as a number, as in how many milliseconds have passed since a universally recognized start time. This may seem complicated, but this is actually quite a popular way to track time as a number, rather than a string. We will introduce this concept much later in the course, but we’ll treat time as a string for our purposes today.

Identifying Boolean Values

Next, you would like to consider any booleans needed for this web application. Here are few that come to mind:

  • Is a user logged in? If so, then maybe they can join a club or edit one they created
  • Is a club currently active?

Identifying Undefined Values

Finally, you want to mention the possibility of handling a few undefined values. Let’s say a student creating a club is unsure about some of the meeting information for that club, but still would like to post it so it could gain some popularity. So, you might provide the option of that value temporarily being “null” as a placeholder, until the student can update the meeting information.
